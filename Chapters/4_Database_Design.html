<p>&nbsp;</p> <h2><strong>What is Database Design?</strong></h2> <ul> <li>Database Design is a collection of processes that facilitate the designing, development, implementation and maintenance of enterprise data management systems</li> </ul> <p>It helps produce &nbsp;database systems</p> <ol> <li>That meet the requirements of the users</li> <li>Have high performance.</li> </ol> <p>The main objectives of database designing are to produce logical and physical designs models of the proposed database system.</p> <p>&nbsp;The logical model concentrates on the data requirements and the data to be stored independent of physical considerations. It does not concern itself with how the data will be stored or where it will be stored physically.</p> <p>&nbsp;The physical data design model involves translating the logical design of the database onto physical media using hardware resources and software systems such as database management systems (DBMS).</p> <h2><strong>Why Database Design is Important ?</strong></h2> <p>Database designing is crucial to&nbsp;<strong>high performance</strong>&nbsp;database system.</p> <p>&nbsp;Apart from improving the performance, properly designed database are easy to maintain, improve data consistency and are cost effective in terms of disk storage space.</p> <p>Note , the genius of a database is in its design . Data operations using SQL is relatively simple</p> <h2><strong>Database development life cycle</strong></h2> <p><a href="https://www.guru99.com/images/DatabaseDesignProcess(1).png"><img alt="Database Design Tutorial: Learn Data Modeling" src="https://www.guru99.com/images/DatabaseDesignProcess(1).png" /></a></p> <p>&nbsp;</p> <p>The database development life cycle has a number of stages that are followed when developing database systems.</p> <p>The steps in the development life cycle do not necessary have to be followed religiously in a sequential manner.</p> <p>On small database systems, the database system development life cycle is usually very simple and does not involve a lot of steps.</p> <p>In order to fully appreciate the above diagram, let&#39;s look at the individual components listed in each step.</p> <h2><strong>Requirements analysis</strong></h2> <ul> <li><strong>Planning</strong>&nbsp;- This stages concerns with planning of entire Database Development Life Cycle &nbsp;It &nbsp;takes into consideration the Information Systems strategy of the organization.</li> <li><strong>System definition</strong>&nbsp;- This stage defines the scope and boundaries of the proposed database system.</li> </ul> <h2><strong>Database designing</strong></h2> <ul> <li><strong>Logical model</strong>&nbsp;- This stage is concerned with developing a database model based on requirements. The entire design is on paper without any physical implementations or specific DBMS considerations.</li> <li><strong>Physical model</strong>&nbsp;- This stage implements the logical model of the database taking into account the DBMS and physical implementation factors.</li> </ul> <h2><strong>Implementation</strong></h2> <ul> <li><strong>Data conversion and loading</strong>&nbsp;- this stage is concerned with importing and converting data from the old system into the new database.</li> <li><strong>Testing</strong>&nbsp;- this stage is concerned with the identification of errors &nbsp;in the newly implemented system .It checks the database against requirement specifications.</li> </ul> <h2><strong>Two Types of Database Techniques</strong></h2> <ol> <li>&nbsp; &nbsp;&nbsp;<strong>Normalization</strong></li> <li>&nbsp; &nbsp;&nbsp;<strong>ER Modeling</strong></li> </ol> <h2><strong>What is Normalization?</strong></h2> <p>Normalization is a database design technique which organizes tables in a manner that reduces redundancy and dependency of data.</p> <p>It divides larger tables to smaller tables and links them using relationships.</p> <h2><strong>What is ER Modeling?</strong></h2> <p><strong>Entity Relationship Modeling&nbsp;</strong>(ER Modeling) is a graphical approach to database design. It uses Entity/Relationship to represent real world objects.</p> <p>An&nbsp;<strong>Entity&nbsp;</strong>is a thing or object in real world that is distinguishable from surrounding environment. For example each employee of an organization is a separate entity. Following are some of major characteristics of entities.</p> <ul> <li>An entity has a set of properties.</li> <li>Entity properties can &nbsp;have values.&nbsp;</li> </ul>
